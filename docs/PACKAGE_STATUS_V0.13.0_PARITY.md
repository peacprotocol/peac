# v0.13.0 Package-Status Parity Audit

Companion to [`docs/PACKAGE_STATUS.md`](./PACKAGE_STATUS.md). This document
records the per-export parity audit used to decide posture for deprecated
and archived packages at v0.13.0. It is hand-maintained; the generator at
`scripts/generate-surface-status.mjs` does not produce it.

**Scope:** v0.13.0 only. The table is frozen at v0.13.0 tag time and
revisited at the next release. If any `Equivalent?` cell moves to `yes`
before the next release opens, it is a pre-condition for retirement.

---

## Why this audit exists

The v0.13.0 release keeps `@peac/disc@0.13.0` published as a one-release
deprecated compatibility alias because `@peac/cli` (published) and
`apps/api` depend on it via `workspace:*`. Removing `@peac/disc` from
`scripts/publish-manifest.json` while published packages depend on it
would produce a `@peac/cli@0.13.0` tarball with an unsatisfiable
`@peac/disc@0.13.0` dependency: release-breaking. Installability beats
surface-count optics.

Before a later release retires `@peac/disc` from the workspace and
publish manifest, every export currently used by internal consumers MUST
have a verified equivalent in `@peac/policy-kit` (or another canonical
public surface). This audit is the migration checklist.

### Current compatibility coverage

The following items must be true in the current repo before `@peac/disc` can be removed from the workspace and publish manifest:

1. **Port `discover()` (and the `MAX_BYTES` / `WELL_KNOWN_PATH` / `DEFAULT_TIMEOUT_MS` constants it relies on) to a canonical public surface.** Preferred home: `@peac/policy-kit` gets the constants + a fetch-injection-shaped `loadPolicyDocumentRemote(options)` that accepts a caller-supplied SSRF-aware fetch. Explicit note: `@peac/policy-kit` remains a pure parser package; remote fetch is done by the caller-supplied fetch, never by the package itself (consistent with the post-v0.12.14 no-network-in-parsers invariant).
2. **Port the tolerant-stripping behavior** of `parse()` (legacy `verify:` / `public_keys:` / `jwks:` line handling + `PEAC_LEGACY_PEAC_TXT_KEY_FIELD` structured DeprecationWarning) into `@peac/policy-kit` under a stable name (e.g. `parsePolicyDocumentTolerant`).
3. **Migrate active internal consumers:**
   - `packages/cli/src/commands/discover.ts` → `@peac/policy-kit.loadPolicyDocumentRemote` (or equivalent).
   - `packages/cli/src/lib/conformance/validators.ts` → `@peac/policy-kit.parsePolicyDocumentTolerant`.
   - `apps/api/src/verifier.ts` → either `@peac/policy-kit.loadPolicyDocumentRemote` or fold into the `/v1/verify` handler's existing issuer-config resolver (this migration may be coupled with the legacy `/verify` handler rewire).
4. **Confirm `validate()` consumers are migrated** to `@peac/policy-kit.validatePolicy` (minor; boolean signature).
5. **Run pack + install smoke** (`pnpm pack --filter @peac/cli`; `tar -xOf *.tgz package/package.json | jq .dependencies`) to confirm no published package depends on `@peac/disc` at retirement tag time.

If step 1 is not completed, `@peac/disc` must remain published as a compatibility alias for longer.
