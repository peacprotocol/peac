# v0.12.13 release notes

Per-release notes that complement `CHANGELOG.md`. The CHANGELOG is the authoritative record of what shipped; this file carries any additional material (links, published artifacts) that is useful alongside the release.

## Release mode and automation

v0.12.13 ships via the two-step `next` to `latest` mode. The publish workflow stages a GitHub Release at tag time and finalizes it after the dist-tag promotion. A post-release truth reconciler verifies npm dist-tags, the git tag, the GitHub Release state, `docs/releases/facts.json`, `docs/releases/current.json`, and `REPO_SURFACE_STATUS.json`.

## Distribution listings

Distribution listing status is tracked in [`docs/case-studies/distribution-submissions.md`](../case-studies/distribution-submissions.md). Each row records the artifact reference and current submission state (`prepared`, `submitted`, `discoverable`).
