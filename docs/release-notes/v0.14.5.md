# v0.14.5 — Public Verification Readiness

PEAC v0.14.5 is a documentation-alignment release. It brings public
verification, security, stability, compatibility, and machine-readable
documentation up to the v0.14.4 release state. It records measured
local SLO baselines for `issue()` and `verifyLocal()`, clarifies the
supported-version table, classifies retired compatibility packages
consistently, refreshes the LLM-facing surface, and removes wording
that could imply PEAC controls, governs, or enforces runtime behavior.

Wire format unchanged. Signing envelope unchanged. Public schema
unchanged. Public API unchanged. CLI surface unchanged. Registered
extension groups (19), receipt types (61), conformance sections (32),
conformance requirement IDs (290), parity-corpus families (11), and
published package count (36) are all unchanged. No new package, no new
type URI, no new conformance ID, no runtime behavior change.

## Highlights

- Measured local SLO baselines stamped against the v0.14.4 machine
  profile in `docs/SLO.md` for `issue()` and `verifyLocal()`;
  reference verifier and MCP round-trip rows are marked explicitly as
  not included in this measured baseline.
- Supported-version table in `SECURITY.md` updated to list `v0.14.x`
  as Active (current), with concrete maintenance windows for
  `v0.13.x`, `v0.12.x`, and `v0.11.x` and an unchanged archival receipt
  format row.
- `@peac/disc` and `@peac/pref` are now formally `archived` in
  `docs/STABILITY-CONTRACT.md` and `docs/COMPATIBILITY_MATRIX.md`,
  consistent with the workspace state (source under `archive/`,
  absent from the publish manifest).
- `docs/TRUST-ARTIFACTS.md` no longer claims CLI execution carriers
  and lifecycle observation records are forthcoming; both shipped in
  v0.14.1 and are classified `stable`.
- `llms.txt` updated to records-first wording, canonical
  `npx -y @peac/mcp-server` install command, and vendor-neutral
  MCP-client phrasing.

## Stability and archive classification

`@peac/disc` and `@peac/pref` are now formally `archived` in
`docs/STABILITY-CONTRACT.md` and `docs/COMPATIBILITY_MATRIX.md`. The
classification matches the existing workspace state: both packages'
source already lives under `archive/discovery/` and `archive/pref/`
respectively, neither is present in `scripts/publish-manifest.json`,
and `REPO_SURFACE_STATUS.json` plus `docs/SURFACE_STATUS.md` already
list both with `archived` state. Replacement paths:

- `@peac/disc` → `@peac/policy-kit` for `peac.txt` policy-document
  loading.
- `@peac/pref` → `@peac/mappings-content-signals` for AIPREF /
  robots.txt / tdmrep parsing.

Historical npm versions of both packages remain installable; new
integrations MUST NOT depend on either.

## Stable identifiers

No protocol identifier change. The reference identifiers below remain
frozen and unchanged from v0.14.4:

- Wire format `interaction-record+jwt` (Wire 0.2) and the archival
  receipt format path.
- `PEAC-Receipt` HTTP header.
- `receipt_jws` and `receipt_ref` MCP `_meta` keys
  (`org.peacprotocol/receipt_jws`, `org.peacprotocol/receipt_ref`).
- `peac-receipt/0.1` (Wire 0.1 archival identifier).
- All 19 extension-group namespaces and all 61 receipt-type URIs.
- The `@peac/protocol` `issue()`, `verify()`, `verifyLocal()` public
  API.

## Boundary

PEAC v0.14.5 does not change the protocol's boundary. PEAC continues
to record portable signed interaction records across organizational,
vendor, and runtime boundaries. It does not orchestrate, settle,
authorize, enforce policy, score trust, manage workflows, or host
runtime infrastructure. The release aligns public
documentation with what has shipped, what is current, what is
archived, and what the verifier actually does.
